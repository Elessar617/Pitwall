# Pitwall v0.3.0 — Strategy Mini-Game

> Phase-4 release. The final pillar: a strategy mini-game played alongside a replayed race — commit a tyre + pit plan pre-race, react to pit-window prompts mid-race, score yourself against the driver's actual outcome.

## What's new

- **Game core** (`pitwall.game`, pure/clock-free): a frozen `StrategyPlan` model with loud validation; a tick-fold (`advance`) detecting pit windows (one-lap warning + lap fallback) and pit events with nearest-planned-lap attribution; the single pinned scoring formula (tyre matches, positional pit-lap accuracy, decision match/miss, classification proximity — total always equals the component sum); a five-line score panel renderer.
- **Plan wizard**: key-driven driver → tyres → pit laps → predicted finish flow with incremental validation and exact render contracts.
- **Strategy screen** (`g` key): the wizard gates the replay — no tick is consumed before the plan commits; pit-window prompts answered with `1`/`2` (the visible prompt is always the decision queue head); the final panel scores the plan against the actual outcome extracted from the replay's stint/pit/position data.

## Known limitations

- **Replay-only**: the game requires `--replay <fixtures-dir>`; live-mode play is deferred (plan-commit-before-tick-0 conflicts with live discovery/backfill ordering — design carried).
- Plans are in-memory (no persistence between launches).
- Scoring ground truth derives from the committed 60 s replay window (single-driver fixtures); full-race scoring awaits multi-driver captures via `scripts/capture_openf1_session.py`.
