# Pitwall v0.2.0 — Live Timing + Track Map

> Phase-3 release. The live pillar: an OpenF1 data layer with a deterministic replay engine, a timing-tower screen with a braille track-position map, and a genuinely-live mode — all reachable from the v0.1.0 season tracker via the `l` key.

## What's new

- **OpenF1 package** (iter08): typed models for seven record kinds behind the frozen coercion boundary; an async client encoding the three lab-proven constraints (percent-encoded operators in parameter *names*, 404-as-empty-window, bounded 429 backoff); typed error hierarchy.
- **Replay engine** (iter08): pure deterministic merge (381 events from the committed 78 KB Montreal excerpt), sleep-injected tick iterator with catch-up seeds and speed multiplier; `--replay <dir> --replay-speed N`.
- **Live timing screen** (iter08): 6-column timing tower with machine-pinned literals, replay states, chassis isolation.
- **Track-position map** (iter09): location parsing + `LocationFeed`, a braille projection/outline/overlay pipeline (20×32 grid derived from the car's own trace), split tower(48)+map(32×20) layout with tower-only fallback.
- **Live mode** (iter10): `--live` session discovery (`session_key=latest`), the `LiveSource` polling source behind the shared `TickSource` protocol (data-head playhead, per-stream cursors with a 241 s laps overlap, second-precision dedupe, per-request error containment, graceful degradation), the multi-driver capture script, and the operator checklist `ship/live-validation-checklist.md`.

## Known limitations

- **Live validation is operator-run**: the live path is verified against mocked transports; the first real-session run follows `ship/live-validation-checklist.md`. OpenF1 location data may lag a session by days — the map may legitimately never appear live.
- **Single-driver location fixtures**: the committed excerpt covers driver 1 only (capture-script extension ships in this release for multi-driver post-session captures).
- A lap left uncompleted longer than 241 s loses its completion event in live mode (recorded in-code).
- Fixed-width 80×24 split layout (48+32); responsive widths deferred.
- Carried review LOWs tracked in `build/workflows/10-live-mode/04-output/OUTPUT.md`.
