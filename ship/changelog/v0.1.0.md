# Pitwall v0.1.0 — Season Tracker

> First release. The complete F1 season-tracker TUI: `uv run pitwall` launches a Textual app with four data screens over a cache-first Jolpica data layer.

## What's in it

- **Season data layer** (iter01): eight frozen domain models with a loud-rejection parse boundary (typed coercion, non-empty identity validation, optional session blocks incl. `SprintShootout`, UTC normalization); async Jolpica client (six endpoints, bounded 429 backoff, pagination guard, typed errors); id-keyed SQLite cache with season-membership scoping and rollback-safe transactional scope refreshes; cache-first `SeasonStore` with calendar-keyed staleness, legitimate-empty-scope caching, and stale-serve-on-failure.
- **App chassis** (iter02): `PitwallApp` with screen registry, `s/n/r/p/q` navigation, once-per-launch season worker, subtitle status contract (`loading…` / `data as of HH:MM UTC` / `· stale` / `load failed`), owned-vs-injected resource lifecycle; `--version` headless; `--refresh-interval` validation (min 5).
- **Schedule screen** (iter03): 22-round calendar table, next-race cursor, loading/empty/error/stale states; the pinned snapshot-delivery pattern (app reactives + watch).
- **Standings screen** (iter04): driver + constructor championships stacked in one 80×24 viewport, position-text-verbatim rendering.
- **Results screen** (iter05): rounds list driving an on-demand, chassis-isolated round-results worker with per-round caching; pre-season guard.
- **Profiles screen** (iter06): driver/constructor rosters via a one-shot roster worker with per-scope partial-failure handling.

## Known limitations

- **Live data**: none yet — live timing, track map, and the strategy game are Phases 3–4 (v0.2.0/v0.3.0).
- `--offline` flag deferred (the store's stale-serve is the degraded path; flag plumbing awaits a cache-touching iteration).
- Staleness is keyed to session *start*, not results publication (SPEC-01 R2): mid-session fetches self-heal at the next session or the 24 h TTL.
- Single-connection SQLite (SPEC-01 R8); `data/pitwall.db` is CWD-relative (no XDG path yet).
- Carried technical debt (tracked in `build/workflows/06-profiles-screen/04-output/OUTPUT.md`): `cache/db.py` split due before the next cache-touching iteration; `position_text` present-but-null parser follow-up; results-screen stale-warning re-notify; private-attr test probes.
